# Runbooks

Use this directory for narrow, command-focused procedures when they are easier to maintain as standalone runbooks.

The primary operational documentation lives under `docs/operations/`. Prefer updating those pages for normal access, maintenance, backups, upgrades, and troubleshooting.

Standalone runbooks should be short and repeatable, with commands, expected output, rollback notes, verification, dependencies, and links back to the relevant operations or setup page.
